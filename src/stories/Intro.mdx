# Twilight Design System:

Power Pop! Theme:

These are the main UI components and patterns used for Please Please Please!.
It is a work in progress. The goal is to expand on it and use it for other projects.

## What's Inside

This design system is organized into several main component categories:

### Dashboard Components

Components used in the creator dashboard for managing queues, approvals, and tasks:

- **Sidebar** - Navigation component with sections and links
- **Approval Panel** - Interface for reviewing and approving ticket submissions
- **Task Module** - Display and interaction for individual tasks
- **Table Components** - Data tables for queue management
- **Earnings Panel** - Financial overview and Stripe integration

### Checkout & Forms

Components for ticket submission and payment processing:

- **Checkout Donation** - Tip input and priority queue selection
- **Ticket Approval Cards** - Display and approval interface for tickets
- **Payment Wrappers** - Stripe integration components

### General Components

Reusable UI primitives and utilities:

- **Button Base** - Foundation button component
- **Tag Base** - Tag/badge component for labels
- **Modals** - Various modal dialogs (About, Legal, Privacy, etc.)

### Background & Effects

- **Hands Background** - Animated background component with WebGL shader effects

## Getting Started

Browse the component categories in the sidebar to explore available components. Each component includes:

- **Stories** - Interactive examples showing different states and variations
- **Autodocs** - Automatically generated documentation with props and usage
- **Controls** - Interactive controls to test component behavior
- **Accessibility** - A11y testing results and guidelines

## Component Organization

Components follow a hierarchical naming structure:

- `Components/[Category]/[Subcategory]/[ComponentName]`

For example:

- `Components/Dashboard/Sidebar/SideBar`
- `Components/General/ButtonBase`
- `Components/Checkout/TicketApprovalCard`

## Development Guidelines

- All components use TypeScript
- Components follow React best practices with proper prop types
- Styling uses Tailwind CSS with custom design tokens
- Components are designed to be reusable and composable
- Most components include comprehensive Storybook stories
